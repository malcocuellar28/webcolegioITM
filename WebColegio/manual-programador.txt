MANUAL DEL PROGRAMADOR - WEB COLEGIO

Este archivo explica de forma rapida que partes del proyecto se pueden editar,
que bloques se duplican y que datos cambian con mas frecuencia.


1. ARCHIVOS PRINCIPALES

- index.html
  Contiene el contenido principal del sitio: textos, cards, secciones, enlaces,
  imagenes, formularios y estructura general.

- css/styles.css
  Contiene colores, temas, espaciados, botones, tooltips, animaciones y estilos.

- js/script.js
  Contiene la logica del menu, temas, contador regresivo, FAQ, toasts,
  validacion del formulario, carrusel de testimonios y otros comportamientos.

- js/news-center.js
  Controla el centro de noticias expandible en noticias.html y noticias-pagina-2.html.

- img/
  Contiene imagenes generales del sitio.

- img/background/
  Contiene fondos visuales usados en secciones como el hero.

- img/icons/
  Contiene los iconos SVG personalizados del proyecto.


2. REGLA GENERAL PARA EDITAR SIN ROMPER EL DISENO

Si quieres agregar mas elementos en una seccion:

- Duplica el bloque completo recomendado.
- Cambia solo texto, imagen, icono o fecha.
- No elimines clases como card, aviso-item, gallery-item, etc.
- No cambies la estructura HTML interna si no es necesario.

Mientras se mantenga la estructura, el tema, las animaciones y el responsive
deberian seguir funcionando automaticamente.


3. NUESTRO COLEGIO EN CIFRAS

Ubicacion: index.html

Cada cifra usa esto:

<div class="stat-number" data-target="500">0</div>

Solo cambia el numero de data-target.
No necesitas tocar el 0 interno.

Ejemplo:

<div class="stat-number" data-target="650">0</div>

Si tambien quieres cambiar el texto inferior, modifica:

<div class="stat-label">Estudiantes</div>

7. GALERIA DE FOTOS

Ubicacion: index.html

Puedes agregar todas las imagenes que quieras.
Se integran automaticamente con grid, modal, hover y responsive.

Duplica desde:

<button class="gallery-item" ...>

hasta:

</button>

Estructura:

<button class="gallery-item" type="button" data-src="img/galeria-5.jpg" aria-label="Abrir imagen Evento escolar 5">
    <img src="img/galeria-5.jpg" alt="Evento escolar 5" loading="lazy">
    <div class="gallery-overlay">
        <svg class="gallery-overlay-icon" ...></svg>
        <div class="gallery-overlay-copy">
            <span class="gallery-overlay-kicker">Categoria</span>
            <span class="gallery-overlay-text">Descripcion breve de la imagen</span>
        </div>
    </div>
</button>

Importante:
- data-src y src deben apuntar a una imagen real existente.
- La primera imagen del HTML se muestra como imagen principal de la galeria.

11. NOTICIAS

Ubicacion principal: index.html
Centro de noticias: noticias.html y noticias-pagina-2.html

En la pagina principal puedes agregar noticias resumen.
Duplica desde:

<article class="card noticia-card">

hasta:

</article>

Estructura:

<article class="card noticia-card">
    <span class="badge-new">Nuevo</span>
    <div class="card-image image-rounded">
        <img src="img/noticia-4.jpg" alt="Nueva noticia" loading="lazy">
    </div>
    <div class="card-content">
        <div class="noticia-meta">
            <span class="noticia-tag">Categoria</span>
            <span class="noticia-fecha">[Fecha]</span>
        </div>
        <h3>Titulo de la noticia</h3>
        <p>Descripcion de la noticia.</p>
        <a href="noticias.html" class="news-link"><span>Leer mas</span><img src="img/icons/leerMas.svg" alt="" class="news-link-icon" aria-hidden="true"></a>
    </div>
</article>

Si no quieres que aparezca la etiqueta Nuevo, elimina solo esta linea:

<span class="badge-new">Nuevo</span>

CENTRO DE NOTICIAS:

El centro de noticias funciona como lista expandible.

- noticias.html = pagina 1 con noticias mas recientes
- noticias-pagina-2.html = pagina 2 con noticias mas antiguas

Cada noticia del centro usa esta estructura:

<article class="news-entry">
    <div class="news-entry-summary">
        <div class="news-entry-media">
            <img src="img/noticia.jpg" alt="Titulo visual" loading="lazy">
        </div>
        <div class="news-entry-copy">
            <div class="noticia-meta">
                <span class="noticia-tag">Categoria</span>
                <span class="noticia-fecha">[Fecha]</span>
            </div>
            <h3>Titulo de la noticia</h3>
            <p class="news-entry-intro">Breve introduccion.</p>
            <button type="button" class="news-expand-link" aria-expanded="false">Leer mas</button>
        </div>
    </div>
    <div class="news-entry-body">
        <div class="news-entry-body-inner">
            <p>Contenido completo.</p>
            <p>Mas contenido.</p>
        </div>
    </div>
</article>

Importante:
- Solo el boton Leer mas / Leer menos abre la noticia.
- La clase is-featured hace que una noticia tenga mas protagonismo.
- La clase is-open deja una noticia abierta por defecto.
- Cuando agregues una noticia nueva al centro, ponla arriba en noticias.html.
- Cuando noticias.html tenga muchas, mueve las mas antiguas a noticias-pagina-2.html.


12. CONTACTANOS

Ubicacion: index.html

En esta seccion normalmente solo editas:

- textos dentro de p
- etiquetas en strong
- placeholders del formulario
- enlaces de redes
- enlace de WhatsApp
- enlace de Google Maps
- texto del boton del mapa

Importante:
- Los enlaces principales de contacto llevan a #contacto-info, no al mapa.
- En movil primero aparece el bloque de contacto/formulario y despues el mapa.
- Si editas enlaces internos hacia contacto, usa #contacto-info cuando quieras llevar al usuario al bloque util de contacto.

No hace falta tocar el CSS ni el JS si solo cambias contenido.

14. CONTADOR REGRESIVO - FALTAN / PARA GRADUACION

Ubicacion: js/script.js

Busca esta configuracion:

const COUNTDOWN_EVENTS = [

Desde ahi puedes cambiar:
- subtitle
- message
- fecha
- hora

Ejemplo actual de graduacion:

{
    key: "graduation",
    subtitle: "para la proxima Graduacion",
    message: "Cuenta regresiva para nuestra ceremonia de graduacion.",
    getDate(year) {
        return new Date(year, 10, 30, 15, 30, 0, 0);
    }
}

Importante:
En JavaScript los meses van de 0 a 11.

- 0 = enero
- 1 = febrero
- 2 = marzo
- 3 = abril
- 4 = mayo
- 5 = junio
- 6 = julio
- 7 = agosto
- 8 = septiembre
- 9 = octubre
- 10 = noviembre
- 11 = diciembre

La logica del contador toma automaticamente el evento futuro mas cercano.

16. ICONOS SVG

Ubicacion: img/icons/

Muchos iconos se cargan desde archivos SVG, pero algunos importantes ahora estan inline.

Inline SVG actuales:
- iconos de tema claro y oscuro
- icono de pantalla completa en la galeria

Si quieres cambiar un icono:

- si es icono externo: sube el nuevo SVG a img/icons/ y cambia la ruta del src en index.html
- si es icono inline: reemplaza el codigo <svg> directamente en el HTML


17. TOOLTIPTEXT / TOOLTIPS

Ubicacion HTML: index.html
Ubicacion CSS: css/styles.css

Para agregar un tooltip a un elemento:

- agrega class="has-tooltip"
- agrega data-tooltip="Tu texto"
- manten aria-label si el elemento es solo icono

Ejemplo:

<button class="has-tooltip" data-tooltip="Cambiar tema" aria-label="Cambiar tema"></button>


18. TEMAS CLARO Y OSCURO

Ubicacion: css/styles.css

Los colores principales estan en variables CSS.
Si cambias contenido HTML y mantienes las clases, el tema se adapta solo.


19. HERO PRINCIPAL

Ubicacion HTML: index.html
Ubicacion estilos: css/styles.css
Ubicacion logica: js/script.js

El hero tiene:
- una imagen principal que cambia automaticamente
- una tarjeta secundaria fija para temporadas o promociones
- un mensaje superior (hero-seasonal-note) que ahora tambien cambia con cada imagen principal

La imagen principal se configura en js/script.js aqui:

const HERO_MAIN_SLIDES = [

Cada slide usa esta estructura:

{
    image: "img/hero-slides/tu-imagen.jpg",
    alt: "Descripcion de la imagen",
    note: "Mensaje corto que aparece arriba en la franja estacional"
}

Importante:
- La pagina no puede leer automaticamente el contenido de una carpeta en una web estatica.
- Si agregas nuevas imagenes al hero, debes subirlas a img/hero-slides/ y luego agregarlas manualmente en HERO_MAIN_SLIDES.
- Actualmente el hero principal usa 4 slides.
- La tarjeta secundaria del hero no rota; se edita directamente en index.html.


20. SI ALGO SE VE MAL, REVISA ESTO PRIMERO

- Que no hayas eliminado una clase CSS importante.
- Que la imagen o SVG exista en la ruta indicada.
- Que hayas duplicado el bloque completo y no solo una parte.
- Que no hayas roto la jerarquia de cierres HTML.


21. RECOMENDACION FINAL

Para cambios de contenido frecuentes, normalmente solo necesitas editar index.html.
Usa css/styles.css solo si quieres cambiar apariencia.
Usa js/script.js solo si quieres cambiar comportamiento.
