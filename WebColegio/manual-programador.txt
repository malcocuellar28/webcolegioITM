MANUAL DEL PROGRAMADOR - WEB COLEGIO

Este manual resume la estructura actual del proyecto y que partes se editan con mas frecuencia sin romper el diseno.

1. ARCHIVOS PRINCIPALES

- index.html
  Pagina principal. Contiene hero, nosotros, FAQ, niveles, carreras, galeria, contador, avisos, eventos, testimonios, noticias resumen, contacto y footer.

- noticias.html
  Centro de noticias, pagina 1. Muestra las noticias mas recientes en formato expandible.

- noticias-pagina-2.html
  Centro de noticias, pagina 2. Archivo de noticias anteriores.

- css/styles.css
  Estilos globales, responsive, tema oscuro, hero, noticias, footer, galeria, FAQ, contacto y componentes reutilizables.

- js/script.js
  Logica de la pagina principal: menu, tema, hero, breadcrumb, scroll, FAQ, modal de galeria, testimonios, toasts, loader, formulario, contador y botones compartir.

- js/news-center.js
  Logica del centro de noticias: abrir/cerrar noticias, sincronizar alturas y mantener enlaces con el tema actual.


2. CARPETAS IMPORTANTES

- img/news/
  Imagenes usadas en noticias.

- img/gallery/
  Imagenes usadas en galeria.

- img/hero-slides/
  Imagenes del slider principal del hero.

- img/icons/
  Iconos SVG del proyecto.

- img/background/
  Fondos decorativos.

- img/frame/
  Recursos decorativos del footer.

- fonts/
  Fuentes locales. El centro de noticias usa Inter-News y PlayfairDisplay-News.


2.1 TIPOGRAFIAS DEL PROYECTO

- Noto Sans
  Texto general del sitio.

- Merriweather Sans
  Titulos institucionales y encabezados de seccion.

- Fraunces
  Hero y ahora tambien titulos promocionales del modal/publicidad.

- Plus Jakarta Sans
  Elementos UI, labels, botones y ahora texto de apoyo publicitario.

- Space Grotesk
  Contadores y bloques numericos.

- Inter News / Playfair Display News
  Uso exclusivo del centro de noticias.

3. REGLA GENERAL PARA EDITAR

- Duplica bloques existentes en lugar de inventar nueva estructura.
- Cambia solo texto, imagen, icono, fecha o etiqueta cuando sea posible.
- No elimines clases como card, gallery-item, news-entry, faq-item, career-card, etc.
- Si cambias rutas de imagen, asegúrate de que el archivo exista de verdad.


4. HERO

Ubicacion: index.html + js/script.js + css/styles.css

- Las imagenes y textos rotativos del hero se configuran en HERO_MAIN_SLIDES dentro de js/script.js.
- La card secundaria del hero se edita manualmente en index.html.
- El slider del hero usa dos imagenes internas para el deslizamiento. No cambies ids como heroMainImage, heroMainImageNext o heroMainDots.


5. SOBRE NOSOTROS

Ubicacion: index.html

- El texto introductorio, mision y vision se editan directo en HTML.
- El video institucional se cambia actualizando el src del iframe.


6. PREGUNTAS FRECUENTES

Ubicacion: index.html

- Duplica un bloque completo .faq-item para agregar otra pregunta.
- La logica del acordeon ya esta resuelta en js/script.js.
- No cambies la estructura interna de faq-question y faq-answer.


7. NIVELES EDUCATIVOS

Ubicacion: index.html

La oferta actual mostrada es:

- Tercer Ciclo
  Con mini-cards para 7mo, 8vo y 9no.

- Bachillerato
  Card general para educacion media sin entrar al detalle de carreras.

Importante:
- La card de Tercer Ciclo usa .nivel-card-grouped, .nivel-subgrid y .nivel-mini-card.
- Si cambias ese bloque, respeta esa estructura para no romper el layout.


8. CARRERAS TECNICAS

Ubicacion: index.html

Cada carrera usa esta estructura base:

<div class="career-card">
    <div class="career-icon"><img src="img/icons/tu-icono.svg" alt="" aria-hidden="true"></div>
    <span class="career-badge">Area</span>
    <h3>Nombre de la carrera</h3>
    <p>Texto comercial principal.</p>
    <ul class="career-points">
        <li>Punto destacado</li>
    </ul>
    <p class="career-highlight">Cierre comercial.</p>
    <span class="career-duration">2 anos o 3 anos</span>
</div>

Debajo de la grilla existe un CTA con:

- .careers-cta-bubble
- .careers-cta
- .careers-cta-actions

El bot del CTA usa img/icons/bot.svg.


9. GALERIA

Ubicacion: index.html

- La primera imagen usa .gallery-item-featured.

Ubicacion: index.html + js/script.js

16. MODAL PROMOCIONAL CONFIGURABLE

Ubicacion HTML: index.html
Ubicacion logica: js/script.js

El modal promocional ya esta preparado pero apagado por defecto.

Busca esta configuracion:

const PROMO_MODAL_CONFIG = {

Campos clave:

- enabled
  true = mostrar modal
  false = no mostrar modal

- version
  Cambia este valor cuando quieras forzar una nueva campaña aunque el usuario ya haya visto la anterior.

- delayMs
  Tiempo de espera antes de mostrarlo.

- showOnceEveryDays
  Cada cuantos dias puede volver a mostrarse al mismo usuario.

- startDate / endDate
  Ventana de fechas en la que la promoción debe estar activa.

- kicker
- title
- text
- image
- imageAlt
- primaryLabel
- primaryHref
- secondaryLabel
- points

- secondaryLabel

Tipografia recomendada para publicidad local:

- Titulos promocionales: --font-promo-title = Fraunces
- Texto de apoyo y botones: --font-promo-body = Plus Jakarta Sans

Estas variables viven en :root dentro de css/styles.css y se usan en:

- .promo-modal-copy h3
- .promo-modal-copy p
- .promo-modal-points
- .promo-modal-actions

points acepta una lista de frases cortas para mostrar beneficios o destacados dentro del modal.

Para editar una campaña normalmente cambias solo:

- enabled
- version
- startDate / endDate
- kicker
- title
- text
- points
- image
- primaryLabel
- primaryHref
- secondaryLabel

El modal se cierra con:

- boton X
- boton secundario
- clic fuera del cuadro
- tecla Escape

Si quieres activarlo en una campaña futura, normalmente basta con:

1. poner enabled en true
2. ajustar fechas
3. actualizar version
4. cambiar texto, imagen y enlace principal

19. LIMPIEZA Y ORDEN

- Antes de borrar un archivo, comprueba si aparece referenciado en HTML, CSS o JS.
- No borres imagenes como primaria.jpg, secundaria.jpg o preescolar.jpg sin revisar noticias y niveles.
- Si agregas nuevos recursos, intenta guardarlos en la carpeta tematica correcta.


20. RECOMENDACION FINAL

Si solo vas a editar contenido:

- trabaja primero en index.html o noticias.html
- evita tocar js/script.js o css/styles.css si no necesitas cambiar comportamiento o diseno

Si vas a cambiar estructura:

- duplica bloques ya existentes
- conserva clases e ids principales
- prueba Home, Noticias, movil y modo oscuro despues de cada cambio
